# 넌 테토야 에겐이야 Layout Structure

## Screen Flow

`input -> reveal -> result`

## Input

```text
┌──────────────────────────────────────┐
│ Header: 넌 테토야 에겐이야             │
│ - 정체성 결 실험                      │
│ - 연속 방문 스트릭                     │
├──────────────────────────────────────┤
│ Name card                             │
│ - 이름 입력                            │
│ - 날짜 기반 변동 안내 문구             │
├──────────────────────────────────────┤
│ Question card #1 (2지선다)             │
├──────────────────────────────────────┤
│ Question card #2 (2지선다)             │
├──────────────────────────────────────┤
│ Question card #3 (2지선다)             │
├──────────────────────────────────────┤
│ Error line (조건 미충족 시)             │
├──────────────────────────────────────┤
│ CTA                                   │
│ - 오늘 성향 분석하기                    │
└──────────────────────────────────────┘
```

## Reveal

```text
┌──────────────────────────────────────┐
│ 분석 리빌 카드                         │
│ - 회전 로더                            │
│ - "지층 결을 해석하고 있어요" 문구      │
│ - 1.7초 후 자동 결과 전환              │
└──────────────────────────────────────┘
```

## Result

```text
┌──────────────────────────────────────┐
│ Today axis card                       │
│ - 중심축(테토/에겐/밸런스)             │

│ - 축 슬라이더(좌 테토 / 우 에겐)        │
├──────────────────────────────────────┤
│ Layer cross-section                    │
│ - 표층 / 중층 / 심층 / 핵               │
│ - 레이어별 색조/결 설명                │
├──────────────────────────────────────┤
│ Premium unlock card (AD)              │
│ - AD 배지                              │
│ - "광고 시청 후 열람" 고지             │
│ - 심층 단면 열기 버튼                  │
├──────────────────────────────────────┤
│ Deep report (광고 해제 후)              │
│ - 심층 해석 3줄                         │
├──────────────────────────────────────┤
│ Recent records                         │
│ - 최근 결과 4개                         │
├──────────────────────────────────────┤
│ CTA                                   │
│ - 다시 분석하기                         │
└──────────────────────────────────────┘
```

## Differentiation Notes

- Palm-lifeline이 `home -> capture -> result` 촬영 흐름인 반면, teto-egen은 `input -> reveal -> result` 심리 축 해석 흐름으로 구성.
- Palm-lifeline의 카메라/권한 중심 상호작용 대신, teto-egen은 3문항 선택 + 날짜 시드 변동 분석으로 반복 사용 동기를 설계.
- Palm-lifeline의 점수 패널/결과 재촬영 CTA 대신, teto-egen은 "지층 단면" 메타포와 AD 게이트 심층 리포트를 핵심 확장 포인트로 사용.

## 차별화 체크리스트 (workflow 기준)

| 항목 | teto-egen 설계 | palm-lifeline 대비 차이 | 판정 |
|------|----------------|-------------------------|------|
| 등급 표현 방식 | 테토/에겐/밸런스 축 | 티어 라벨 기반 결과 카드 | 다름 |
| 점수 시각화 | 축 포인터 + 레이어 메타포 | progress bar 점수 패널 | 다름 |
| 데이터 카테고리 | 이름+3문항+날짜 시드 | 촬영 이미지+로컬 계산 점수 | 다름 |
| 조언/팁 표현 | 레이어별 문장 + 심층 3줄 | 해석 리스트 + 추천 액션 리스트 | 다름 |
| 프리미엄 영역 | 심층 단면 리포트 AD 해제 | 재촬영/재분석 중심 CTA | 다름 |
| 전체 화면 흐름 | 입력 -> 리빌 -> 결과 | 홈 -> 촬영 -> 결과 | 다름 |
| 고유 컴포넌트 | 지층 단면 4계층 카드 | 촬영 프레임 + 권한 카드 | 다름 |

통과 기준: 7개 중 7개 차별화.
